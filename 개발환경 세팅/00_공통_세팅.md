# 🛠 공통 개발환경 세팅 가이드

이 문서는 My-Bo 프로젝트의 전체적인 개발 환경을 구축하기 위한 공통 필수 도구들을 설명합니다.

## 1. 필수 소프트웨어 설치

### ☕ Java & Kotlin
- **JDK 17**: Spring Boot 프로젝트(`auth-server`, `gateway`) 실행을 위해 필수입니다.
  - 추천: Amazon Corretto 17 또는 Oracle OpenJDK 17
- **IDE**: IntelliJ IDEA (Community 또는 Ultimate) 추천

### 🟢 Node.js
- **Node.js (LTS)**: `frontend` 및 `scraper-server` 실행을 위해 필요합니다.
  - 버전: v20.x 이상 권장
- **Package Manager**: `npm` (Node 설치 시 함께 설치됨)

### 🗄 데이터베이스 (Docker 사용 권장)
프로젝트 실행을 위해 MySQL과 MongoDB가 필요합니다. Docker를 사용하면 간편하게 설치할 수 있습니다.

#### MySQL (for auth-server)
- **Port**: 3306
- **Database**: `member` (반드시 미리 생성해야 함)
- **User/PW**: `root` / `비밀번호` (또는 각 프로젝트의 `.env`에서 설정)

#### MongoDB (for scraper-server)
- **Port**: 27017
- **ConnectionString**: `mongodb://localhost:27017/scraper`

---

## 2. 보안 및 환경 설정 (.env)
대부분의 서버 프로젝트는 보안상의 이유로 데이터베이스 비밀번호, API 키 등을 코드에 직접 기록하지 않고 `.env` 파일을 통해 관리합니다.

- **주의**: `.env` 파일은 보안 정보를 포함하므로 절대 Git에 커밋하지 않습니다. (이미 `.gitignore`에 포함되어 있습니다.)
- **공통 절차**: 각 서버 폴더에 있는 `.env.example` (혹은 가이드 문서의 예시)을 참고하여 `.env` 파일을 생성하고 자신의 환경에 맞게 수정하세요.

---

## 3. 전체 프로젝트 구조 및 흐름
1. **Frontend (Nuxt 3)**: 사용자가 접속하는 화면 (Port 3000)
2. **Gateway (Spring Cloud)**: 모든 API 요청의 관문 (Port 8080)
3. **Auth-Server (Spring Boot)**: 회원가입/로그인 및 인증 처리 (Port 9001)
4. **Scraper-Server (NestJS)**: 데이터 크롤링 및 수집 (Port 4000)

사용자의 요청은 다음과 같이 흐릅니다:
`Frontend` -> `Gateway` -> (`Auth-Server` or `Scraper-Server`)
