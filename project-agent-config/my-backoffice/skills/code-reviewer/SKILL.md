# Code Reviewer Skill

Gemini 또는 서브에이전트가 구현한 코드를 체계적으로 리뷰하는 스킬.

## 사용 시점
- Gemini가 구현을 완료했을 때
- be-agent 또는 fe-agent가 리뷰를 위임받았을 때

## 리뷰 프로세스

### 1. 구현 범위 파악
- 핸드오프 문서의 완료 기준 체크리스트 확인
- 변경된 파일 목록 파악

### 2. 코드 리뷰 항목

#### 공통 체크리스트
```
[ ] 완료 기준 체크리스트 모두 충족
[ ] 기존 아키텍처 패턴 준수
[ ] 보안 취약점 없음 (SQL 인젝션, XSS, 인증 우회 등)
[ ] 에러 핸들링 적절
[ ] 불필요한 코드/의존성 없음
[ ] 하드코딩된 값 없음 (환경변수 사용)
```

#### 백엔드 추가 체크리스트
```
[ ] 헥사고날 아키텍처 레이어 경계 준수 (Spring Boot)
[ ] 비즈니스 로직이 도메인/유스케이스 레이어에 위치
[ ] 컨트롤러는 얇게 유지 (위임만)
[ ] DTO로 API 경계 분리
[ ] 트랜잭션 처리 적절
[ ] DB 인덱스 고려
[ ] JWT 검증 누락 없음
```

#### 프론트엔드 추가 체크리스트
```
[ ] Composition API 사용 (<script setup lang="ts">)
[ ] 공유 상태는 컴포저블 외부에 정의
[ ] 적절한 레이아웃 사용 (default/dashboard)
[ ] 로딩/에러 상태 처리
[ ] 반응형 디자인 (Tailwind)
[ ] onUnmounted에서 사이드 이펙트 정리
```

### 3. 리뷰 결과 출력 형식

```markdown
## 코드 리뷰 결과: {기능명}

### 요약
- 완료 기준 달성: {N}/{M}
- 전반적 평가: PASS / PASS_WITH_COMMENTS / FAIL

### 상세 피드백

#### 필수 수정 (FAIL 사유)
1. {파일경로}:{라인} — {문제} → {해결책}

#### 권장 개선 (선택적)
1. {파일경로}:{라인} — {제안}

#### 잘된 점
- {항목}
```

### 4. 리뷰 완료 후
- PASS: 핸드오프 파일을 `docs/handoff/reviewed/`로 이동 + `-final` 추가
- FAIL: 피드백을 핸드오프 파일 Q&A 섹션에 QUESTIONS로 추가
