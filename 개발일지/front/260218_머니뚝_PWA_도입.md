## 3. 트러블슈팅

### [TS-1] Manifest 미노출
- **증상**: `@vite-pwa/nuxt` 모듈 적용 후 브라우저 DevTools > Application > Manifest에 아무것도 표시되지 않음
- **원인**: Nuxt 4 SSR 모드에서 `@vite-pwa/nuxt`가 `<link rel="manifest">` 태그를 HTML `<head>`에 자동 주입하지 않는 문제
- **해결**: `nuxt.config.ts`의 `app.head.link`에 수동으로 manifest 링크 추가
  ```ts
  link: [
    { rel: 'manifest', href: '/manifest.webmanifest' },
    { rel: 'apple-touch-icon', href: '/pwa-192x192.png' },
  ]
  ```
- **참고**: `manifest.webmanifest`, `sw.js`, `workbox-*.js` 파일은 `nuxt build` 시 `.output/public/`에 정상 생성됨. dev 모드(`npm run dev`)에서는 `devOptions: { enabled: true }` 설정 필요

---

### [TS-2] `$pwa.showInstallPrompt` 반응성 미작동
- **증상**: `@vite-pwa/nuxt`의 `useNuxtApp().$pwa.showInstallPrompt`를 `v-if`에 사용했으나 버튼이 노출되지 않음
- **원인**: `$pwa` 플러그인 객체의 반응성이 Vue 템플릿에서 제대로 추적되지 않는 타이밍 이슈
- **해결**: `$pwa` 대신 `window.addEventListener('beforeinstallprompt', ...)` 이벤트를 직접 핸들링하는 `usePwaInstall.ts` composable 직접 구현
- **핵심 포인트**: `beforeinstallprompt`는 브라우저당 한 번만 발생하므로 모듈 레벨 싱글턴으로 관리해야 함

---

### [TS-3] 카카오톡 인앱 브라우저에서 설치 버튼 미노출
- **증상**: 카카오톡 공유 링크로 진입 시 `beforeinstallprompt`가 발생하지 않아 설치 버튼이 안 보임
- **원인**: 카카오톡은 인앱 브라우저(WebView)로 링크를 열며, WebView는 `beforeinstallprompt` 이벤트를 지원하지 않음
- **해결**: UA로 인앱 브라우저 감지 후 Android intent URL로 Chrome을 직접 실행
  ```ts
  // UA 감지
  isInAppBrowser = /KAKAOTALK|Instagram|FBAN|FBAV|Line\/|NAVER|Snapchat/i.test(ua)

  // Chrome 강제 실행
  intent://{url}?autoinstall=1#Intent;scheme=https;package=com.android.chrome;end
  ```

---

### [TS-4] Chrome으로 이동 후 설치 버튼 미노출 (autoInstall 모드 버그)
- **증상**: 카카오톡에서 "Chrome으로 열기" 후 Chrome에서도 설치 버튼이 안 뜸
- **원인**: `autoInstall` 모드에서 `beforeinstallprompt` 핸들러를 `return`으로 조기 종료해 `canInstall`을 `true`로 세팅하지 않았던 버그
- **해결**: 모든 경우에 `e.preventDefault()` + `deferredPrompt` 저장. `autoInstallMode && canInstall` 조건일 때 하단 설치 모달 자동 노출
- **UX 플로우**:
  ```
  카카오톡 → "Chrome으로 열기" 탭
    → Chrome이 ?autoinstall=1로 실행
    → beforeinstallprompt 감지 → canInstall = true
    → 설치 모달 자동 팝업 → "설치하기" 탭 1번 → 완료
  ```
- **한계**: `prompt()`는 브라우저 보안 정책상 반드시 유저 제스처 안에서 호출해야 하므로 탭 0번(완전 자동)은 불가

### [TS-5] Chrome 진입 및 뒤로가기 시 흰 화면 깜빡임
- **증상**: 카카오톡 → Chrome 이동 시, 그리고 앱 내 뒤로가기 시 흰 화면이 잠깐 보임
- **원인**:
  - intent URL로 Chrome이 새 Activity로 콜드 스타트 → 이전 페이지 없어 Paint Holding 미동작
  - `manifest.theme_color: '#ffffff'` 설정으로 Chrome이 CSS 로드 전에 흰색을 칠함
  - `body`에만 배경색 지정 → `html` 요소는 여전히 흰색
- **해결**:
  ```ts
  // nuxt.config.ts
  manifest: {
    theme_color: '#0f172a',      // 추가
    background_color: '#0f172a', // 추가
  }
  // head meta
  { name: 'theme-color', content: '#0f172a' } // 추가
  ```
  ```css
  /* app.vue */
  html, body { background-color: #0f172a; } /* html 추가 */
  ```
- **직접 입력 vs intent 차이**: Chrome 주소창 직접 입력은 기존 탭에서 내비게이션이므로 Paint Holding이 동작 → 깜빡임 없음. intent URL은 새 Activity 콜드 스타트 → Paint Holding 없음 → 깜빡임 발생

---

### [TS-6] PWA 앱 종료 시 흰 화면 깜빡임
- **증상**: PWA 설치 후 앱 종료(홈 버튼/제스처) 시 흰 화면이 잠깐 보임
- **원인**: Android Activity 종료 애니메이션 중 Chrome 내부 Activity Window 배경색이 노출됨. 이는 Chrome 앱 자체의 Android 테마에서 오는 것으로 웹 표준으로 접근 불가
- **대응 범위**:
  - `manifest.background_color: '#0f172a'` → 앱 실행 스플래시에는 적용됨 ✅
  - Activity 종료 애니메이션 배경 → **웹에서 제어 불가** ❌
- **결론**: 네이티브 앱 수준의 완전한 제어는 React Native / Flutter 등 네이티브 래퍼 필요. 현재 구현 범위 내에서는 최선
