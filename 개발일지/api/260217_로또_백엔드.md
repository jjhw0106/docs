### ✅ 1. 로또 서비스 백엔드 (`lotto-server`) 신규 구축
**목표:** 사용자의 로또 구매 내역 관리 및 당첨 확인을 자동화하는 백엔드 서버를 구축한다.

*   **헥사고날 아키텍처 적용**:
    *   도메인 모델(`WinningNumbers`, `PurchaseGame`)과 외부 기술(JPA, 동행복권 API)을 분리하여 유지보수성 향상.
    *   입출력 포트 인터페이스 정의를 통한 계층 간 결합도 완화.
*   **핵심 도메인 로직 구현**:
    *   `RankCalculator`: 6개 번호와 보너스 번호를 대조하여 1등~5등 및 낙첨을 판정하는 순수 도메인 로직 구현.
    *   `RoundCalculator`: 2002-12-07 기준 현재 날짜에 맞는 로또 회차 자동 계산 기능 구현.
*   **외부 API 연동**:
    *   동행복권 공식 API를 호출하여 회차별 당첨 번호, 추첨일, 당첨금 정보를 수집하는 클라이언트 구현.

### ✅ 2. 인증 체계 개선 및 데이터 통합
*   **JWT 클레임 확장 (`auth-server`)**:
    *   기존 Access Token에 `memberId`가 누락되어 서비스를 넘나드는 데이터 식별이 어렵던 문제를 해결.
    *   `JwtTokenProvider`를 수정하여 토큰 생성 시 `memberId`를 클레임에 추가하고, 로또 서버에서 이를 추출하여 사용자별 구매 내역을 관리할 수 있게 함.
*   **API Gateway 라우팅 설정**:
    *   `gateway`에 `/lotto/**` 경로를 추가하여 프론트엔드가 일관된 엔드포인트로 접근 가능하도록 설정.
